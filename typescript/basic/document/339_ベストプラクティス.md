# #339 「ベストプラクティス」

四国めたん「nullable型のベストプラクティスについて学びましょう!」
ずんだもん「型エイリアスを使うと便利なんだね!」
四国めたん「はい。Nullable<T> という型を定義すると、コードが読みやすくなります。」
ずんだもん「型ガードも推奨されるの?」
四国めたん「その通りです。厳密等価演算子でnullチェックすると、型が自動的に絞り込まれます。」
ずんだもん「Optional Chainingも重要だね!」
四国めたん「はい。?. 演算子で、nullチェックを簡潔に書けます。」
ずんだもん「ベストプラクティスを守って、安全なコードを書くのだ!」

---

## 📺 画面表示用コード

```typescript
/** Example 1: 型エイリアス */
type Nullable<T> = T | null;
function findUser(id: number): Nullable<User> {
  return users.find(u => u.id === id) ?? null;
}

/** Example 2: 型ガード */
if (user !== null) {
  console.log(user.name); // 安全
}

/** Example 3: Optional Chaining */
const name = user?.name ?? "Guest";
const city = user?.address?.city;
```

# #349 「ベストプラクティス」

四国めたん「undefinedableのベストプラクティスを学びましょう!」
ずんだもん「オプショナルプロパティを積極的に使うんだね!」
四国めたん「はい。port?: number のように、必須でない値はオプショナルにします。」
ずんだもん「デフォルト値はどう設定するの?」
四国めたん「Nullish Coalescing演算子 ?? を使って、config.port ?? 8080 のように設定します。」
ずんだもん「Optional Chainingも組み合わせるの?」
四国めたん「その通りです。data?.process()?.value ?? defaultValue のように連鎖できます。」
ずんだもん「ベストプラクティスで読みやすいコードを書くのだ!」

---

## 📺 画面表示用コード

```typescript
/** Example 1: オプショナルの活用 */
interface Config {
  host: string;
  port?: number;        // オプショナル
  timeout?: number;     // オプショナル
}

/** Example 2: デフォルト値 */
function connect(config: Config) {
  const port = config.port ?? 8080;
  const timeout = config.timeout ?? 3000;
}

/** Example 3: Optional Chaining */
const result = data?.process()?.value ?? defaultValue;
```

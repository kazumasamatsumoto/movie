# #276 「間違い(1) - ==使用」

四国めたん「==演算子の問題点について学びましょう!」
ずんだもん「==は型変換するから、予期しない動作が起きるんだね!」
四国めたん「はい。userInput == 0だと、'0'や空文字列もtrueになってしまいます。」
ずんだもん「value == nullは、nullとundefinedを区別しないから曖昧だよね?」
四国めたん「その通りです。必ず===を使って厳密な比較をしましょう。」
ずんだもん「userInput === 0なら、数値の0だけがtrueになるんだね!」
四国めたん「value === nullは、nullのみを厳密にチェックできるので安全です。」
ずんだもん「型安全なコードを書くために、==は絶対に避けるのだ!」

---

## 📺 画面表示用コード

```typescript
// ❌ 間違い: ==の使用
if (userInput == 0) {  // '0'や空文字列もtrue
  // 予期しない動作
}

if (value == null) {  // nullとundefinedを区別しない
  // 曖昧な処理
}
```

```typescript
// ✅ 正しい: ===の使用
if (userInput === 0) {  // 数値の0のみtrue
  // 意図通りの動作
}

if (value === null) {  // nullのみ
  // 明確な処理
}
```

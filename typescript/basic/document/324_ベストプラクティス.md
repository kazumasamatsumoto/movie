# #324 「ベストプラクティス」

四国めたん「strictNullChecksのベストプラクティスを学びましょう!」
ずんだもん「明示的な型定義が大切なんだね!」
四国めたん「はい。戻り値の型をUser | nullと明示的に指定します。」
ずんだもん「型ガードを使うのも重要だよね?」
四国めたん「その通りです。nullチェックで安全にプロパティアクセスします。」
ずんだもん「Nullish Coalescingも便利なんだね!」
四国めたん「はい。??演算子でデフォルト値を簡潔に指定できます。」
ずんだもん「ベストプラクティスに従って、安全なコードを書くのだ!」

---


```typescript
/** Example 1: 明示的な型定義 */
function getUser(id: number): User | null {
  return users.find(u => u.id === id) ?? null;
}

/** Example 2: 型ガード */
if (user !== null) {
  console.log(user.name); // 安全
}

/** Example 3: Nullish Coalescing */
const name = user?.name ?? "Guest";
const port = config.port ?? 3000;
```

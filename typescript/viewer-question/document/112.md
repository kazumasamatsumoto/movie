# #112 「`triple-slash directives`見かけたらどうする？」制作ドキュメント

## 台本
ずんだもん「`triple-slash directives`見かけたらどうする？」
四国めたん「v5.9では`triple-slash directives`の必要性が大幅に減って、モダンなモジュールシステムが推奨されています」
ずんだもん「ぼくは古いコードで`/// <reference path="..." />`を見たら、まずモジュール化を検討するんだよね」
四国めたん「`triple-slash directives`は型定義ファイルの依存関係を示しますが、ES Modulesで代替できることが多いです」
ずんだもん「レガシーコードの移行プロジェクトで、この記法を消していくのが定番タスクになってるんだよ！」
四国めたん「みなさんは`triple-slash directives`をまだ使っていますか？」
ずんだもん「移行の経験があったらコメントで教えてね！」

## 画面表示用コード

```typescript
// 古い書き方（triple-slash directive）
/// <reference path="./types.d.ts" />

// モダンな書き方（ES Modules）
import type { MyType } from './types';
```

```typescript
// 型定義ファイルでのリファレンス（まだ必要なケース）
/// <reference types="node" />

declare namespace NodeJS {
  interface ProcessEnv {
    NODE_ENV: 'development' | 'production';
  }
}
```

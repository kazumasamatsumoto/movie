## 台本
ずんだもん「`Parameters`って実務で使う機会ある？」
四国めたん「v5.9では`Parameters`は関数の引数型を抽出するユーティリティで、関数のラッパーやテストで活躍します」
ずんだもん「ぼくは既存関数をラップする時や、モック関数の型を作る時に`Parameters`をよく使うんだよね」
四国めたん「`Parameters`を使うと、関数の引数型を重複して書かずに再利用でき、型の一貫性が保たれます」
ずんだもん「APIクライアントのラッパー関数で、元の関数の引数型を`Parameters`で取得して型安全に書けたんだよ！」
四国めたん「みなさんは`Parameters`を実務で使っていますか？」
ずんだもん「使用ケースや便利な場面をコメントで教えてね！」

## 画面表示用コード

```typescript
// 関数のラッパーを作成
function apiCall(endpoint: string, options: RequestOptions): Promise<Response> {
  // ...
}

// Parametersで引数型を再利用
function apiCallWithLogging(...args: Parameters<typeof apiCall>) {
  console.log('Calling API:', args[0]);
  return apiCall(...args);
}
```

```typescript
// モック関数の型を作成
type MockFunction<T extends (...args: any) => any> = {
  (...args: Parameters<T>): ReturnType<T>;
  calls: Array<Parameters<T>>;
};

function createMock<T extends (...args: any) => any>(
  impl: T
): MockFunction<T> {
  const calls: Array<Parameters<T>> = [];
  const mock = ((...args: Parameters<T>) => {
    calls.push(args);
    return impl(...args);
  }) as MockFunction<T>;
  mock.calls = calls;
  return mock;
}
```

```typescript
// 引数の一部を固定した関数を作成
function partial<T extends (...args: any) => any>(
  fn: T,
  ...fixedArgs: Partial<Parameters<T>>
) {
  return (...remainingArgs: any[]) =>
    fn(...[...fixedArgs, ...remainingArgs]);
}
```

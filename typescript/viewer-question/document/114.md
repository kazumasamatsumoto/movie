# #114 「`@ts-expect-error`と`@ts-ignore`どっち使う？」制作ドキュメント

## 台本
ずんだもん「`@ts-expect-error`と`@ts-ignore`どっち使う？」
四国めたん「v5.9では`@ts-expect-error`が推奨されていて、エラーが存在しない場合に警告が出るようになっています」
ずんだもん「ぼくは必ず`@ts-expect-error`を使って、将来的にエラーが解消されたら気づけるようにしてるんだよね」
四国めたん「`@ts-expect-error`は意図的にエラーを抑制していることが明確で、コードレビューでも理由が追跡しやすいです」
ずんだもん「一時的な型エラー回避で`@ts-ignore`を使ってたプロジェクトで、後から問題を見つけるのが大変だったんだよ！」
四国めたん「みなさんはどちらを使っていますか？」
ずんだもん「使い分けのルールがあったらコメントで教えてね！」

## 画面表示用コード

```typescript
// 推奨: @ts-expect-error（エラーがなくなると警告される）
// @ts-expect-error: 外部ライブラリの型定義が不完全
const result = externalLib.missingType();

// 非推奨: @ts-ignore（エラーが解消されても気づけない）
// @ts-ignore
const result2 = externalLib.missingType();
```

```typescript
// エラーが存在しない場合の挙動
// @ts-expect-error: Unused '@ts-expect-error' directive.
const valid: string = 'hello'; // エラーがないので警告される
```

## 制作メモ
- **問いの切り口**: @ts-expect-errorと@ts-ignoreの使い分け
- **想定視聴者層**: 型エラー抑制の適切な方法を知りたいエンジニア
- **押さえる技術キーワード**: @ts-expect-error, @ts-ignore, エラー抑制
- **キャラの立場**: 四国めたん=ベストプラクティス推奨 / ずんだもん=実体験視点
- **コメントで聞きたいこと**: どちらを使っているか、使い分けのルール
- **補足メモ**: @ts-expect-errorの優位性を具体的に説明

## 画面表示用コード

```typescript
// コーディング規約の例
/**
 * 型パラメータ命名規則:
 * - 単一パラメータ: T
 * - 複数パラメータ: T, U, K, V (順不同)
 * - 意味が必要な場合: Tプレフィックス + 意味のある名前
 *   例: TData, TError, TRequest, TResponse
 */

// 統一された例
interface Repository<TEntity, TId = string> {
  findById(id: TId): Promise<TEntity | null>;
}

type AsyncResult<TData, TError = Error> =
  | { status: 'success'; data: TData }
  | { status: 'error'; error: TError };
```

```typescript
// ESLintルールでの強制（カスタムルール）
// .eslintrc.js
{
  "rules": {
    "naming-convention": [
      "error",
      {
        "selector": "typeParameter",
        "format": ["PascalCase"],
        "prefix": ["T"]  // Tプレフィックスを強制
      }
    ]
  }
}
```

```typescript
// ドキュメント化された命名パターン
/**
 * プロジェクト固有の型パラメータ命名:
 * - TEntity: エンティティ型
 * - TDto: Data Transfer Object型
 * - TError: エラー型
 * - TId: ID型
 */
```

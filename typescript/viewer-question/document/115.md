# #115 「`@ts-nocheck`使ったことある？推奨してる？」制作ドキュメント

## 台本
ずんだもん「`@ts-nocheck`使ったことある？推奨してる？」
四国めたん「v5.9では`@ts-nocheck`は移行期の一時的な措置として認識されていて、長期使用は推奨されていません」
ずんだもん「ぼくはJavaScriptからTypeScriptへの段階的移行で、どうしても対応できないファイルに使ったことがあるんだよね」
四国めたん「`@ts-nocheck`はファイル全体の型チェックを無効化するので、型安全性が失われることを理解する必要があります」
ずんだもん「大規模移行プロジェクトで一時的に使ったけど、必ずチケットを切って後で対応するルールにしてたんだよ！」
四国めたん「みなさんは`@ts-nocheck`を使ったことがありますか？」
ずんだもん「使用ケースや代替案があったらコメントで教えてね！」

## 画面表示用コード

```typescript
// @ts-nocheck
// 移行期の一時的な措置としてのファイル全体の型チェック無効化
function legacyCode() {
  const x = unknownAPI(); // 型エラーが無視される
  return x.anything.goes; // すべての型チェックがスキップされる
}
```

```typescript
// より良い代替案: 段階的に型を追加
function migratingCode() {
  // @ts-expect-error: 移行中 - Issue #123で対応予定
  const x = unknownAPI();
  return x as any; // 意図的な箇所だけを明示
}
```

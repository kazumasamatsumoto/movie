# #131 「`ReadonlyArray<T>`と`readonly T[]`どっち派？」制作ドキュメント

## 台本
ずんだもん「`ReadonlyArray<T>`と`readonly T[]`どっち派？」
四国めたん「v5.9では両方の記法が完全に同等で、どちらを使っても同じ型として扱われます」
ずんだもん「ぼくは短い`readonly T[]`の方が読みやすくて好きだけど、プロジェクトで統一するのが大事だと思うんだよね」
四国めたん「`readonly T[]`の方が簡潔ですが、`ReadonlyArray<T>`の方がジェネリック型であることが明確です」
ずんだもん「コードレビューで両方混在してたから、ESLintルールで`readonly T[]`に統一したんだよ！」
四国めたん「みなさんはどちらの記法を使っていますか？」
ずんだもん「統一のルールや好みをコメントで教えてね！」

## 画面表示用コード

```typescript
// 両方は完全に同等
type Users1 = ReadonlyArray<User>;
type Users2 = readonly User[];

// 同じように使える
function process1(users: ReadonlyArray<User>): void { }
function process2(users: readonly User[]): void { }
```

```typescript
// readonly T[] の方が簡潔
function sum(numbers: readonly number[]): number {
  return numbers.reduce((a, b) => a + b, 0);
}

// ReadonlyArray<T> の方がジェネリック型として明確
type Matrix<T> = ReadonlyArray<ReadonlyArray<T>>;
```

```typescript
// ESLintで統一を強制
// @typescript-eslint/array-type
{
  "rules": {
    "@typescript-eslint/array-type": ["error", {
      "default": "array",  // T[] を優先
      "readonly": "array"  // readonly T[] を優先
    }]
  }
}
```

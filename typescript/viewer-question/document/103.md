# #103 「`branded types`導入してる？それとも不要派？」制作ドキュメント

## 台本
ずんだもん「`branded types`導入してる？それとも不要派？」
四国めたん「v5.9の型システムでは`branded types`がより安定して使えるようになりました」
ずんだもん「ぼくはUserIdとProductIdを区別するために`branded types`を使うのが好きなんだよね」
四国めたん「`branded types`を使えば、構造的に同じ型でも名目的に区別できます」
ずんだもん「現場だと単なるstringを区別できて、IDの取り違えバグが防げるんだよ！」
四国めたん「みなさんは`branded types`を導入していますか？」
ずんだもん「メリット・デメリットをコメントで教えて！」

## 画面表示用コード

```typescript
type Brand<K, T> = K & { __brand: T };
type UserId = Brand<string, "UserId">;
type ProductId = Brand<string, "ProductId">;

const userId = "user123" as UserId;
const productId = "prod456" as ProductId;
// userId = productId; // エラー！
```

- **想定視聴者層**: 型安全性を高めたいエンジニア
- **押さえる技術キーワード**: branded types, nominal typing
- **キャラの立場**: 四国めたん=整理役 / ずんだもん=実装現場視点
- **コメントで聞きたいこと**: 導入の有無とメリット・デメリット
- **補足メモ**: ID型の取り違え防止の実用例を紹介

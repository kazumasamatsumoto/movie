# #116 「`exactOptionalPropertyTypes`有効にしてる？」制作ドキュメント

## 台本
ずんだもん「`exactOptionalPropertyTypes`有効にしてる？」
四国めたん「v5.9では`exactOptionalPropertyTypes`がstrictモードに含まれ、`undefined`の明示的な扱いが厳密になりました」
ずんだもん「ぼくはこのオプション有効にして、オプショナルと`undefined`の違いを明確にするのが好きなんだよね」
四国めたん「`exactOptionalPropertyTypes`を有効にすると、`{ key?: string }`に`{ key: undefined }`を代入できなくなり、より厳密な型チェックが可能です」
ずんだもん「APIレスポンスの型定義で、存在しないフィールドと明示的な`undefined`を区別できて助かったんだよ！」
四国めたん「みなさんはこのオプションを有効にしていますか？」
ずんだもん「使ってみた感想をコメントで教えてね！」

## 画面表示用コード

```typescript
// exactOptionalPropertyTypes: true の場合
type User = {
  name: string;
  age?: number; // 存在しないか、数値
};

const user1: User = { name: 'Alice' }; // OK
const user2: User = { name: 'Bob', age: 25 }; // OK
const user3: User = { name: 'Charlie', age: undefined }; // Error!
```

```typescript
// undefined を許可したい場合は明示的に書く
type UserWithUndefined = {
  name: string;
  age?: number | undefined; // 存在しない、数値、または明示的なundefined
};

const user4: UserWithUndefined = { name: 'Dave', age: undefined }; // OK
```

# #001 「`interface`派？`type`派？あなたはどっち？」制作ドキュメント

## 台本
ずんだもん「`interface`派？`type`派？あなたはどっち？」
四国めたん「v5.9はユーティリティ型が充実したので`interface`派も改めて選び直す時期ですね」
ずんだもん「ぼくは`interface`派でドメインモデルを書く派だけど、DTOは`type`派でまとめたいんだよね」
四国めたん「`interface`派は宣言マージや継承が得意、一方`type`派はユニオンや条件型が柔軟です」
ずんだもん「現場だと`interface`派と`type`派を巡るレビュー合戦が始まって、命名規則ドキュメントが増刷されるんだよ！」
四国めたん「みなさんはどうしていますか？コメントで“interfaceかtypeか”って教えてください！」
ずんだもん「理由も添えてくれたら嬉しいな！」

## 画面表示用コード

```typescript
interface User {
  id: string;
  name: string;
}

type UserSummary = Pick<User, "id" | "name">;

type ApiResult<T> = { data: T } | { error: string };
```

## 制作メモ
- **問いの切り口**: `interface`派？`type`派？あなたはどっち？
- **想定視聴者層**: TypeScriptの設計やルール決めで悩むエンジニア
- **押さえる技術キーワード**: `interface`, `type alias`, 宣言マージ
- **キャラの立場**: 四国めたん=整理役 / ずんだもん=実装現場の迷いを代弁
- **コメントで聞きたいこと**: interfaceかtypeか
- **補足メモ**: 派閥と使い分けルールをコメントで募る。
- **進行トーン**: 派閥トークを軽快に煽るテンポ

# #117 「`undefined`と`optional`の区別、厳密にしてる？」制作ドキュメント

## 台本
ずんだもん「`undefined`と`optional`の区別、厳密にしてる？」
四国めたん「v5.9では`exactOptionalPropertyTypes`との組み合わせで、`undefined`と`optional`の区別がより重要になっています」
ずんだもん「ぼくはAPIの型定義で、フィールドの省略と明示的な`undefined`を使い分けるようにしてるんだよね」
四国めたん「`optional`は存在しない可能性を示し、`| undefined`は明示的に`undefined`が渡される可能性を示します」
ずんだもん「JSONシリアライズで、省略されたフィールドと`null`の違いを型で表現できるようになって便利だったんだよ！」
四国めたん「みなさんは`undefined`と`optional`を厳密に区別していますか？」
ずんだもん「使い分けのテクニックがあったらコメントで教えてね！」

## 画面表示用コード

```typescript
// optionalプロパティ（存在しないかもしれない）
type Config = {
  host: string;
  port?: number; // 省略可能
};

const config1: Config = { host: 'localhost' }; // OK
const config2: Config = { host: 'localhost', port: 3000 }; // OK
```

```typescript
// undefinedを含むユニオン型（明示的にundefinedを渡せる）
type Settings = {
  theme: 'light' | 'dark' | undefined; // undefinedも明示的に設定可能
};

const settings1: Settings = { theme: 'light' }; // OK
const settings2: Settings = { theme: undefined }; // OK（明示的なundefined）
```

```typescript
// 実用例: APIレスポンス
type UserUpdate = {
  name?: string; // 更新しない場合は省略
  email: string | undefined; // undefinedで明示的にクリア
};
```

# #109 「`union`のタグ付け、どういう命名規則？」制作ドキュメント

## 台本
ずんだもん「`union`のタグ付け、どういう命名規則？」
四国めたん「v5.9では判別可能なユニオン型の設計がより柔軟になりました」
ずんだもん「ぼくは`type`や`kind`をタグ名にするのが好きだけど、`status`を使う人も多いんだよね」
四国めたん「タグ名を統一すればコードの可読性が上がり、パターンマッチが明確になります」
ずんだもん「現場だとチームで命名規則を決めておかないと、レビュー時に混乱が起きるんだよ！」
四国めたん「みなさんはどんな命名規則を使っていますか？」
ずんだもん「タグ名の決め方をコメントで共有して！」

## 画面表示用コード

```typescript
// type派
type EventA = { type: "click"; x: number; y: number };
type EventB = { type: "hover"; elementId: string };

// kind派
type ShapeA = { kind: "circle"; radius: number };
type ShapeB = { kind: "rectangle"; width: number; height: number };
```

## 制作メモ
- **問いの切り口**: unionのタグ付けにどんな命名規則を使っているか
- **想定視聴者層**: TypeScriptの設計規約に関心があるエンジニア
- **押さえる技術キーワード**: discriminated union, タグ名, 命名規則
- **キャラの立場**: 四国めたん=整理役 / ずんだもん=実装現場視点
- **コメントで聞きたいこと**: タグ名の命名規則
- **補足メモ**: type, kind, statusなど様々なタグ名の例を紹介

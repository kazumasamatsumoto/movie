# #107 「`is`と`asserts`どっちが好き？」制作ドキュメント

## 台本
ずんだもん「`is`と`asserts`どっちが好き？」
四国めたん「v5.9では`is`と`asserts`の両方が安定して使えます」
ずんだもん「ぼくは`is`で条件分岐するのが好きだけど、`asserts`で早期リターンする方が読みやすい場面もあるんだよね」
四国めたん「`is`は真偽値を返して型を絞り込み、`asserts`は例外を投げて型を保証します」
ずんだもん「現場だと`asserts`を使うと防御的プログラミングが楽になるんだよ！」
四国めたん「みなさんはどちらが好きですか？」
ずんだもん「好みの理由もコメントで教えてね！」

## 画面表示用コード

```typescript
// is派
function isValidUser(user: unknown): user is { id: string } {
  return typeof user === "object" && user !== null && "id" in user;
}

// asserts派
function assertValidUser(user: unknown): asserts user is { id: string } {
  if (!isValidUser(user)) throw new Error("Invalid user");
}
```

## 制作メモ
- **問いの切り口**: isとassertsのどちらが好きか
- **想定視聴者層**: TypeScriptの型絞り込みに関心があるエンジニア
- **押さえる技術キーワード**: is, asserts, type predicate
- **キャラの立場**: 四国めたん=整理役 / ずんだもん=実装現場視点
- **コメントで聞きたいこと**: どちらが好きか、その理由
- **補足メモ**: 両者のコーディングスタイルの違いを紹介

# #141 「ジェネリクスの命名、`T`以外使う派？」制作ドキュメント

## 台本
ずんだもん「ジェネリクスの命名、`T`以外使う派？」
四国めたん「v5.9では`T`、`U`、`K`、`V`が慣習的に使われますが、意味のある名前を付ける派も増えています」
ずんだもん「ぼくは単純な型には`T`を使うけど、複雑な型や複数のジェネリクスがある場合は`TData`とか意味のある名前を付けるんだよね」
四国めたん「`T`は簡潔ですが、複雑なジェネリクスでは`TUser`、`TResponse`のように意味を持たせた方が理解しやすくなります」
ずんだもん「API型で`T`、`U`、`V`が乱立してたコードを、`TRequest`、`TResponse`、`TError`に変えたら可読性が上がったんだよ！」
四国めたん「みなさんはジェネリクスにどんな命名をしていますか？」
ずんだもん「命名ルールや工夫をコメントで教えてね！」

## 画面表示用コード

```typescript
// シンプルな場合は T, U, K, V
function identity<T>(value: T): T {
  return value;
}

function map<T, U>(items: T[], fn: (item: T) => U): U[] {
  return items.map(fn);
}

type Record<K extends string | number | symbol, V> = { [P in K]: V };
```

```typescript
// 複雑な場合は意味のある名前
interface ApiResponse<TData, TError = Error> {
  data?: TData;
  error?: TError;
  meta: {
    timestamp: number;
  };
}

type AsyncState<TData, TError = Error> =
  | { status: 'loading' }
  | { status: 'success'; data: TData }
```

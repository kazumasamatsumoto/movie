# #133 「`const assertion`の使いどころ、決めてる？」制作ドキュメント

## 台本
ずんだもん「`const assertion`の使いどころ、決めてる？」
四国めたん「v5.9では`as const`の推論がさらに改善され、リテラル型の活用がより便利になっています」
ずんだもん「ぼくは定数オブジェクトや設定値には積極的に`as const`を付けて、厳密な型推論を得るようにしてるんだよね」
四国めたん「`as const`を使うと、リテラル型として推論され、オブジェクトや配列がreadonlyになります」
ずんだもん「ルーティング定義に`as const`を使ったら、パス文字列のtypoがコンパイル時に検出できたんだよ！」
四国めたん「みなさんは`const assertion`をどんな場面で使っていますか？」
ずんだもん「効果的な使い方をコメントで教えてね！」

## 画面表示用コード

```typescript
// as constなし（広い型で推論される）
const config1 = {
  env: 'development', // string型
  port: 3000          // number型
};

// as constあり（リテラル型で推論される）
const config2 = {
  env: 'development', // 'development'型
  port: 3000          // 3000型
} as const;
```

```typescript
// 配列でのas const
const ROUTES = [
  { path: '/home', component: Home },
  { path: '/about', component: About },
] as const;

type RoutePath = typeof ROUTES[number]['path']; // '/home' | '/about'
```

```typescript
// 定数の集合からユニオン型を生成
const STATUSES = ['pending', 'success', 'error'] as const;
type Status = typeof STATUSES[number]; // 'pending' | 'success' | 'error'

function handleStatus(status: Status) {
  // statusは厳密に型付けされる
}
```

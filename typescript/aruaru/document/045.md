# #045 「`eslint --max-warnings 0`でCIが止まる恐怖」制作ドキュメント

## 台本
ずんだもん「TypeScript書いてるとさ、`eslint --max-warnings 0`でCIが止まる恐怖って絶対あるよね？」
四国めたん「わかります。Lintを厳しくするとCIが赤く染まりますよね」
ずんだもん「しかもそのまま`eslint --max-warnings 0`でCIが止まる恐怖状態になって、チーム全員が固まるんだよ！」
四国めたん「そこでTypeScriptのLintと型チェックの住み分けを使えば、`eslint --max-warnings 0`と`tsc --noEmit`を役割分担させましょう」
ずんだもん「いや待って、それやると`eslint --max-warnings 0`でCIが止まる恐怖第2ラウンドが始まるやつ！」
四国めたん「共感した方は“Lint警察”ってコメントで教えてください！」
ずんだもん「あなたのTypeScriptあるあるも募集中だよ！」

## 画面表示用コード

```typescript
// package.json
{
  "scripts": {
    "lint": "eslint ./src --max-warnings=0",
    "typecheck": "tsc --noEmit"
  }
}

// .eslintrc.cjs
module.exports = {
  extends: ["plugin:@typescript-eslint/recommended"],
  rules: {
    "@typescript-eslint/no-explicit-any": "warn"
  }
};

// CI設定イメージ
yarn lint
yarn typecheck
```

## 制作メモ
- **想定シーン**: `eslint --max-warnings 0`でCIが止まる恐怖に遭遇した瞬間を切り取るネタ
- **狙いトーン**: CI担当の叫びを代弁
- **技術キーワード**: ESLint, tsc --noEmit, CI
- **使いたい一言**: 「Lint警察」
- **CTA案**: コメントで「Lint警察」と書いてもらい、同じ経験を募る
- **備考**: Lintルールと型チェッカーの線引きをユーモラスに。

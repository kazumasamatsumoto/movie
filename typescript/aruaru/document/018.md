# #018 「`extends`で条件分岐しすぎて自分も迷子」制作ドキュメント

## 台本
ずんだもん「TypeScript書いてるとさ、`extends`で条件分岐しすぎて自分も迷子って絶対あるよね？」
四国めたん「わかります。tsconfigを触ると一晩溶けるのはあるあるですよね」
ずんだもん「しかもそのまま`extends`で条件分岐しすぎて自分も迷子状態になって、チーム全員が固まるんだよ！」
四国めたん「そこでTypeScriptの`tsconfig`整備を使えば、`extends`や`include`を整理するとビルドが安定します」
ずんだもん「いや待って、それやると`extends`で条件分岐しすぎて自分も迷子第2ラウンドが始まるやつ！」
四国めたん「共感した方は“tsconfig地獄”ってコメントで教えてください！」
ずんだもん「あなたのTypeScriptあるあるも募集中だよ！」

## 画面表示用コード

```typescript
// tsconfig.base.json
{
  "compilerOptions": {
    "strict": true,
    "module": "esnext"
  }
}

// tsconfig.app.json
{
  "extends": "./tsconfig.base.json",
  "include": ["src"],
  "exclude": ["dist"]
}

// package.json
{
  "scripts": {
    "typecheck": "tsc -p tsconfig.app.json --noEmit"
  }
}
```

## 制作メモ
- **想定シーン**: `extends`で条件分岐しすぎて自分も迷子に遭遇した瞬間を切り取るネタ
- **狙いトーン**: 設定ファイルに疲弊しつつも冷静に指南
- **技術キーワード**: tsconfig, extends, compilerOptions
- **使いたい一言**: 「tsconfig地獄」
- **CTA案**: コメントで「tsconfig地獄」と書いてもらい、同じ経験を募る
- **備考**: 設定ファイル迷子ネタで攻めるとコメントが集まります。

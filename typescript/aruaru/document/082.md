# #082 「`readonly string[]`を`string[]`に戻したい葛藤」制作ドキュメント

## 台本
ずんだもん「TypeScript書いてるとさ、`readonly string[]`を`string[]`に戻したい葛藤って絶対あるよね？」
四国めたん「わかります。読み取り専用にした途端 push したくなるんですよね」
ずんだもん「しかもそのまま`readonly string[]`を`string[]`に戻したい葛藤状態になって、チーム全員が固まるんだよ！」
四国めたん「そこでTypeScriptの`Readonly`と`Mutable`の切替を使えば、`Readonly`で守って必要な箇所だけ`Mutable`化できます」
ずんだもん「いや待って、それやると`readonly string[]`を`string[]`に戻したい葛藤第2ラウンドが始まるやつ！」
四国めたん「共感した方は“readonly封印”ってコメントで教えてください！」

## 画面表示用コード

```typescript
type User = { id: string; tags: string[] };

const frozen: Readonly<User> = { id: "1", tags: ["ts"] };

type Mutable<T> = {
  -readonly [K in keyof T]: T[K];
};

const editable: Mutable<typeof frozen> = { ...frozen };
editable.tags.push("js");

const tags: ReadonlyArray<string> = ["ts", "js"];
const copied = [...tags];
copied.push("rust");
```

## 制作メモ

- **技術キーワード**: `Readonly`, `ReadonlyArray`, ミューテーション
- **使いたい一言**: 「readonly封印」
- **CTA案**: コメントで「readonly封印」と書いてもらい、同じ経験を募る
- **備考**: イミュータブル運用のツラさをネタにすると良い感じです。

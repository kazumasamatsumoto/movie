# #088 「`Project References`導入を提案して封鎖される」制作ドキュメント

## 台本
ずんだもん「TypeScript書いてるとさ、`Project References`導入を提案して封鎖されるって絶対あるよね？」
四国めたん「わかります。ワークスペース設定が噛み合わなくて朝になるのはあるあるですよね」
ずんだもん「しかもそのまま`Project References`導入を提案して封鎖される状態になって、チーム全員が固まるんだよ！」
四国めたん「そこでTypeScriptのモノレポ型共有を使えば、`pnpm`や`yarn`のworkspaceで型パッケージを共有しましょう」
ずんだもん「いや待って、それやると`Project References`導入を提案して封鎖される第2ラウンドが始まるやつ！」
四国めたん「共感した方は“workspace迷子”ってコメントで教えてください！」
ずんだもん「あなたのTypeScriptあるあるも募集中だよ！」

## 画面表示用コード

```typescript
# pnpm-workspace.yaml
packages:
  - packages/*
  - apps/*

// packages/types/package.json
{
  "name": "@app/types",
  "version": "1.0.0",
  "types": "index.d.ts"
}

// apps/web/package.json
{
  "dependencies": {
    "@app/types": "workspace:*"
  }
}
```

## 制作メモ
- **想定シーン**: `Project References`導入を提案して封鎖されるに遭遇した瞬間を切り取るネタ
- **狙いトーン**: リードエンジニアの嘆き風
- **技術キーワード**: pnpm workspace, モノレポ, 型共有
- **使いたい一言**: 「workspace迷子」
- **CTA案**: コメントで「workspace迷子」と書いてもらい、同じ経験を募る
- **備考**: 型パッケージをどう配るかの苦労話がハマります。

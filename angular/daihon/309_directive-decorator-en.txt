# #309 「@Directive デコレータ」台本

ずんだもん「What do we configure with the @Directive decorator?」
四国めたん「We declare metadata such as the selector, standalone flag, providers, and host settings.」
ずんだもん「Should we always include the standalone flag?」
四国めたん「In v20 the recommended setup is standalone: true so the Directive does not depend on an NgModule.」
ずんだもん「Can we configure host bindings on the decorator side?」
四国めたん「Yes. Define events and attributes together with the `host` property on @Directive.」
ずんだもん「Can we use the same settings when we scaffold with the Angular CLI?」
四国めたん「Sure. After generation you can add any metadata you need and extend it flexibly.」
